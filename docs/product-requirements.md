# EnvNexus 产品需求文档 (PRD)

> **文档版本**：v1.0
> **产品名称**：环枢 (EnvNexus)
> **文档状态**：已冻结（对应 MVP 及 Phase 1-6 核心需求）

---

## 1. 引言

### 1.1 编写目的
本文档旨在明确 EnvNexus 平台的功能需求、非功能需求及用户交互流程，作为研发、测试和产品团队的执行基准。

### 1.2 目标用户与角色
1. **系统管理员 (System Admin)**：负责整个私有化/SaaS 平台的底层运维与 License 管理。
2. **租户管理员 (Tenant Admin)**：企业 IT 负责人，负责配置本企业的策略、模型、生成下载链接和查看全局审计。
3. **IT 支持专家 (Support Expert)**：负责通过平台远程发起诊断和下发修复建议。
4. **终端用户 (End User)**：在自己电脑上安装 Agent，遇到问题时发起诊断并审批修复动作。

## 2. 核心功能需求

### 2.1 多租户与身份管理
- **需求描述**：系统必须支持多租户逻辑隔离。
- **功能点**：
  - 租户的创建、禁用与删除。
  - 基于 RBAC 的五级角色权限控制（Owner, Admin, Operator, Auditor, Viewer）。
  - 支持租户级别的独立配置（模型、策略、品牌资源）。

### 2.2 策略与模型配置 (Profiles)
- **需求描述**：允许租户自定义 AI 引擎和安全边界。
- **功能点**：
  - **Model Profile**：配置大模型提供商（OpenAI, DeepSeek, 内部私有模型等）及 API Key。
  - **Policy Profile**：配置终端安全策略（如：禁止修改注册表、禁止删除系统目录文件），支持 JSON 格式规则。
  - **Agent Profile**：配置客户端心跳频率、离线行为等。

### 2.3 客户端分发与激活
- **需求描述**：提供便捷、安全的客户端下发机制。
- **功能点**：
  - **生成下载链接**：管理员在 Web 端选择系统架构（Windows/macOS/Linux），生成带有过期时间的预签名下载链接。
  - **零编译注入**：下载的安装包必须自动注入当前租户的身份信息。
  - **静默激活**：终端用户安装后，Agent 自动连接平台并完成设备注册，无需用户手动输入 Token。

### 2.4 诊断与修复闭环
- **需求描述**：这是产品的核心业务流，必须严格遵循“只读诊断 -> 审批 -> 修复”的流程。
- **功能点**：
  - **只读诊断**：Agent 提供本地对话框，用户输入问题。Agent 自动调用只读工具（如读取日志、查进程）收集上下文并发送给云端大模型。
  - **生成建议**：大模型返回人类可读的修复建议及待执行的底层脚本。
  - **本地策略拦截**：Agent 接收到脚本后，必须先与本地 Policy 比对，违规操作直接拦截。
  - **用户审批**：合规操作在桌面端弹出“风险警告与审批框”，必须由用户点击“同意”按钮。
  - **安全执行**：用户同意后，Agent 执行脚本，并将结果和输出回传给云端。

### 2.5 审计与合规
- **需求描述**：所有系统变更必须可追溯。
- **功能点**：
  - **全量日志**：记录所有诊断对话、工具调用、审批动作及执行结果。
  - **防篡改**：审计日志写入后不可修改（Append-Only）。
  - **脱敏导出**：支持将审计日志导出为 NDJSON 格式，并自动脱敏 PII（个人隐私信息）。

### 2.6 企业级集成
- **需求描述**：融入企业现有的 IT 生态。
- **功能点**：
  - **Webhook**：支持将高危审批请求、设备离线等事件推送到企业微信、飞书或 Slack。

## 3. 非功能需求

### 3.1 性能与并发指标
- **单实例支撑**：单个平台实例需支持至少 1,000 台设备注册，200 台设备同时在线心跳。
- **并发会话**：支持至少 50 个并发的 WebSocket 诊断会话。
- **分发性能**：客户端安装包的生成与下载链接返回时间必须在 1 秒以内（毫秒级）。

### 3.2 安全要求
- **零信任架构**：云端不保存终端的最高控制权，所有高危操作的最终决定权在终端用户（审批制）。
- **数据隔离**：不同租户的数据（尤其是审计日志和配置）在数据库和对象存储层面必须严格隔离。
- **传输加密**：所有端到云的通信必须走 HTTPS/WSS。

### 3.3 可用性与容灾
- **离线降级**：当 Agent 无法连接云端时，必须能继续提供本地基础诊断功能，并支持导出加密诊断包。
- **组件回退**：当云端 MinIO（对象存储）不可用时，审计日志必须能自动回退写入本地文件系统。

## 4. 用户交互界面 (UI/UX) 核心要求

### 4.1 Console Web (云端控制台)
- 采用现代化的 SaaS 仪表盘设计。
- 必须支持中英双语（i18n）。
- 关键操作（如删除租户、吊销设备）必须有二次确认防呆设计。

### 4.2 Agent Desktop (终端桌面端)
- **极简设计**：平时隐藏在系统托盘，不打扰用户。
- **对话式交互**：主界面采用类似 ChatGPT 的对话流形式，降低用户理解门槛。
- **清晰的审批流**：审批弹窗必须用醒目的颜色（如红色/黄色）标示风险等级，并明确列出即将执行的命令原文。
