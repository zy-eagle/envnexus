> **文档版本**：v1.0
> **日期**：2026-03-31
> **文档状态**：评审中
> **关联设计文档**：`docs/omnidev-state/main/04-design.md` (v2.3)

## 1. 概述

### 1.1 业务背景

EnvNexus 平台当前支持运维人员通过控制台发起远程诊断会话，但缺少**直接向设备下发执行命令**的能力。在实际运维场景中，运维人员经常需要在一台或多台设备上执行特定命令（如重启服务、清理缓存、更新配置），且这些操作需要经过**上级审批**后才能执行，以确保组织管控和操作安全。

### 1.2 核心场景

```
运维人员（王五）需要在 3 台生产服务器上重启 nginx。

1. 王五在控制台创建命令任务，选择目标设备，填写命令
2. 系统根据审批策略，将审批请求发送给运维主管（张三）
3. 张三在控制台"待审批"页面 或 飞书审批卡片中点击"通过"
4. 命令自动下发到 3 台设备执行
5. 执行结果回传，通知王五和张三
```

### 1.3 与现有"本地对话审批"的区别

本模块是**全新的组织级审批系统**，与 agent-desktop 上已有的"本地对话确认"是两套独立机制：

| 维度 | 本地对话确认（已有） | 远程命令审批（本模块） |
|------|---------------------|----------------------|
| **谁触发** | 设备用户自己在 Desktop 对话 | 运维人员通过控制台/IM Bot |
| **谁审批** | 设备用户自己（确认高危操作） | 运维的上级/指定审批人 |
| **审批目的** | 防止 AI 误操作（安全确认） | 组织管控（权限审批） |
| **审批渠道** | Desktop 聊天流内联确认弹窗 | 控制台"待审批"页面 + IM 审批卡片 |
| **设备范围** | 单台（本机） | 单台或批量多台 |
| **本模块是否改动** | 不改动，完全保持现有逻辑 | **本模块设计重点** |

## 2. 角色与权限

### 2.1 涉及角色

| 角色 | 说明 | 典型用户 |
|------|------|---------|
| **运维人员** | 创建命令任务，选择目标设备，填写命令 | 运维工程师、IT 支持专家 |
| **审批人** | 审核运维人员提交的命令任务，通过或拒绝 | 运维主管、技术总监 |
| **管理员** | 配置审批策略、IM 渠道、管理权限 | 租户管理员、IT 负责人 |

### 2.2 权限定义

| 权限标识 | 说明 | 默认角色 |
|---------|------|---------|
| `command:create` | 创建命令任务 | Operator |
| `command:approve` | 审批命令任务 | 审批人角色（由审批策略指定） |
| `command:emergency` | 使用紧急模式（跳过审批） | SRE / 高级运维 |
| `command:bypass_approval` | 免审批特权（所有命令自动通过） | 超级管理员 |
| `policy:manage` | 配置审批策略 | Tenant Admin |
| `integration:manage` | 配置 IM 渠道凭据 | Tenant Admin |

## 3. 功能模块

### 3.1 命令任务管理

#### 3.1.1 创建命令任务

**入口**：控制台 → 侧边栏 → 命令任务 → [+ 新建任务]

**交互流程**：

1. 运维人员点击"新建任务"，弹出创建表单
2. 填写以下字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 任务标题 | 文本 | 是 | 简要描述任务目的，如"重启生产环境 nginx" |
| 命令类型 | 单选 | 是 | Shell 命令 / 工具调用 |
| 命令内容 | 文本 | 是 | 具体命令，如 `systemctl restart nginx` |
| 目标设备 | 多选 | 是 | 从设备列表中勾选，或按标签批量选择 |
| 风险等级 | 单选 | 是 | L1 (低) / L2 (中) / L3 (高)，系统可自动建议 |
| 备注 | 文本 | 否 | 补充说明 |

3. 提交后，系统根据风险等级匹配审批策略：
   - 若策略为**自动通过**（如 L1 低风险）→ 命令立即下发执行
   - 若策略指定了**审批人** → 任务进入"待审批"状态，通知审批人
   - 若未配置策略 → 默认需要租户管理员审批

4. 运维人员可在任务列表中查看任务状态

**表单 UI 示意**：

```
┌─────────────────────────────────────────┐
│  新建命令任务                              │
│                                          │
│  任务标题: [重启生产环境 nginx        ]     │
│                                          │
│  命令类型: ○ Shell 命令  ○ 工具调用         │
│                                          │
│  命令内容: [systemctl restart nginx   ]     │
│                                          │
│  目标设备: [✓] 设备A  [✓] 设备B  [ ] 设备C  │
│            或: 按标签选择 [生产环境 ▼]       │
│                                          │
│  风险等级: ○ L1 低  ● L2 中  ○ L3 高       │
│                                          │
│  审批人: [张三 (运维主管) ▼] (自动匹配策略)  │
│                                          │
│  备注: [紧急修复，nginx 502 错误     ]      │
│                                          │
│  [取消]                    [提交审批]       │
└─────────────────────────────────────────┘
```

**风险等级自动建议规则**：

| 风险等级 | 命令特征 | 示例 |
|---------|---------|------|
| L1 (低) | 只读诊断命令 | `systeminfo`, `df -h`, `ping`, `cat /etc/os-release` |
| L2 (中) | 服务操作 | `systemctl restart nginx`, 清缓存, 重启进程 |
| L3 (高) | 系统级操作 | 修改配置文件, 安装/卸载软件, `docker compose up -d` |

> 系统根据命令内容自动建议风险等级，运维人员可手动调整。

#### 3.1.2 命令任务列表

**入口**：控制台 → 侧边栏 → 命令任务

**功能**：
- 展示当前租户下所有命令任务（运维视角：我创建的任务）
- 支持按状态筛选：全部 / 待审批 / 已通过 / 已拒绝 / 执行中 / 已完成 / 已过期 / 已取消
- 支持按关键词搜索（任务标题、命令内容）
- 每条任务显示：标题、命令、目标设备数、风险等级、状态、创建时间、审批人

**列表项交互**：
- 点击任务 → 进入任务详情页
- "待审批"状态的任务 → 运维人员可点击"取消"撤回

#### 3.1.3 命令任务详情

**入口**：命令任务列表 → 点击某条任务

**展示内容**：

| 区域 | 内容 |
|------|------|
| **基本信息** | 任务标题、命令类型、命令内容、风险等级、创建人、创建时间 |
| **审批信息** | 审批人、审批状态、审批时间、审批备注 |
| **执行结果** | 每台设备一行，显示：设备名称、执行状态、输出内容、退出码、耗时 |

**设备执行状态**：

| 状态 | 图标 | 说明 |
|------|------|------|
| pending | ⏳ | 等待下发 |
| sent | 📤 | 已发送到设备 |
| running | 🔄 | 执行中 |
| succeeded | ✅ | 执行成功 |
| failed | ❌ | 执行失败（显示错误信息） |
| timeout | ⏰ | 执行超时 |
| skipped | ⏭️ | 设备离线，跳过 |

**详情页 UI 示意**：

```

│  命令任务详情                                                     │

### 3.2 待审批（审批人视角）

#### 3.2.1 待审批列表

**入口**：控制台 → 侧边栏 → 待审批（带角标显示待处理数量）

**可见性**：仅拥有 `command:approve` 权限的用户可见此菜单项。

**数据范围**：仅显示当前登录用户作为审批人的、状态为"待审批"的任务。

**交互流程**：

1. 审批人登录控制台后，侧边栏"待审批"入口显示红色角标（如 `3`）
2. 点击进入待审批列表
3. 每条待审批任务显示：
   - 任务标题、命令内容
   - 申请人（运维人员）
   - 目标设备列表
   - 风险等级（L1 绿色 / L2 黄色 / L3 红色）
   - 提交时间（相对时间，如"2 分钟前"）
   - 剩余审批时间（接近超时时显示倒计时警告）
4. 每条任务提供三个操作按钮：
   - **[✅ 通过]** — 弹出确认弹窗，可填写审批备注
   - **[❌ 拒绝]** — 弹出确认弹窗，可填写拒绝原因
   - **[查看详情 →]** — 跳转到任务详情页

**角标更新机制**：
- 页面加载时请求待审批数量
- 每 30 秒轮询一次更新角标数字
- 未来可升级为 WebSocket 实时推送

#### 3.2.2 审批操作

**通过操作**：

1. 审批人点击 [✅ 通过]
2. 弹出确认弹窗：

```
┌─────────────────────────────────────────┐
│  确认通过                                │
│                                          │
│  您即将通过以下命令任务：                  │
│  「重启生产环境 nginx」                    │
│  命令将下发到 3 台设备执行。               │
│                                          │
│  审批备注: [同意执行，注意观察日志    ]     │
│  (选填)                                  │
│                                          │
│  [取消]                    [确认通过]     │
└─────────────────────────────────────────┘
```

3. 确认后：
   - 任务状态变为"已通过"
   - 命令自动下发到目标设备
   - 通知运维人员：任务已通过
   - 待审批列表移除该任务，角标数字 -1

**拒绝操作**：

1. 审批人点击 [❌ 拒绝]
2. 弹出确认弹窗：

```
┌─────────────────────────────────────────┐
│  确认拒绝                                │
│                                          │
│  您即将拒绝以下命令任务：                  │
│  「重启生产环境 nginx」                    │
│                                          │
│  拒绝原因: [请先确认 nginx 配置无误   ]    │
│  (选填)                                  │
│                                          │
│  [取消]                    [确认拒绝]     │
└─────────────────────────────────────────┘
```

3. 确认后：
   - 任务状态变为"已拒绝"
   - 命令不会执行
   - 通知运维人员：任务已被拒绝（附带拒绝原因）

#### 3.2.3 审批超时

- 每条审批任务有超时时间（由审批策略配置，默认 30 分钟）
- 接近超时时（剩余 5 分钟），列表项显示橙色倒计时警告
- 超时后：
  - 任务状态自动变为"已过期"
  - 通知运维人员：审批已超时
  - 运维人员可重新提交任务

### 3.3 审批策略管理

#### 3.3.1 功能说明

审批策略定义了**什么风险等级的命令需要谁来审批**。这是审批人配置的核心入口。

**入口**：控制台 → 侧边栏 → 审批策略

**权限**：`policy:manage`（管理员）

#### 3.3.2 策略列表

展示当前租户下所有审批策略，每条策略显示：

| 列 | 说明 |
|----|------|
| 策略名称 | 如"中风险运维主管审批" |
| 风险等级 | L1 / L2 / L3 / *（所有） |
| 审批人 | 指定用户名 或 角色名 |
| 自动通过 | 是 / 否 |
| 审批超时 | N 分钟 |
| 优先级 | 数字（越大越优先） |
| 操作 | [编辑] [删除] |

#### 3.3.3 新建/编辑策略

**表单字段**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 策略名称 | 文本 | 是 | 策略的描述性名称 |
| 风险等级 | 单选 | 是 | L1 (低) / L2 (中) / L3 (高) / * (所有等级) |
| 审批方式 | 单选 | 是 | 指定审批人 / 按角色审批 |
| 审批人 | 下拉选择 | 条件必填 | 选择"指定审批人"时，从用户列表中选择 |
| 审批角色 | 下拉选择 | 条件必填 | 选择"按角色审批"时，从角色列表中选择（该角色下任一用户通过即可） |
| 自动通过 | 开关 | 否 | 开启后跳过审批，命令直接执行（适用于 L1 低风险） |
| 审批超时 | 数字 | 是 | 超时时间（分钟），超时后任务自动过期。默认 30 |
| 优先级 | 数字 | 是 | 同一风险等级有多条策略时，优先级高的先匹配。默认 0 |

**交互规则**：
- 选择"自动通过"时，审批人/角色字段隐藏（不需要审批人）
- 同一风险等级可配置多条策略，按优先级从高到低匹配，匹配到第一条即停止
- 删除策略时弹出二次确认

#### 3.3.4 策略匹配逻辑

```
运维创建命令任务（风险等级 = L2）
  ↓
查找审批策略（租户 + 风险等级 = L2，按优先级降序）
  ↓
├── 找到策略，auto_approve = true
│     → 自动通过，命令直接下发
├── 找到策略，指定审批人 = 张三
│     → 张三成为审批人，IM 通知 + 控制台待审批可见
├── 找到策略，指定角色 = 运维主管
│     → 运维主管角色下所有用户都是审批人，任一通过即可
└── 未找到 L2 策略 → 查找 * (通配) 策略
      └── 仍未找到 → 默认需要租户管理员审批
```

#### 3.3.5 默认策略建议

系统初始化时，建议为租户预置以下默认策略：

| 策略名称 | 风险等级 | 审批人 | 自动通过 | 超时 |
|---------|---------|--------|---------|------|
| 低风险自动通过 | L1 | — | ✅ 是 | — |
| 中风险需审批 | L2 | 租户管理员 | ❌ 否 | 30 分钟 |
| 高风险需审批 | L3 | 租户管理员 | ❌ 否 | 60 分钟 |

管理员可根据实际组织架构修改审批人。

### 3.4 紧急模式（逃生通道）

#### 3.4.1 业务场景

在生产事故（如服务宕机、安全漏洞）等紧急情况下，运维人员需要**立即执行命令**，不能等待审批流程。系统提供"紧急模式"作为逃生通道。

#### 3.4.2 触发方式

| 触发方式 | 操作 | 所需权限 |
|---------|------|---------|
| 控制台 | 新建任务时勾选"🚨 紧急执行" | `command:emergency` |
| 飞书 Bot | 使用 `/exec!` 命令（感叹号表示紧急） | `command:emergency` |

#### 3.4.3 交互流程

**控制台紧急模式**：

1. 运维人员在新建任务表单中勾选"🚨 紧急执行（跳过审批，直接下发）"
2. 勾选后，表单变化：
   - 出现警告提示："⚠️ 紧急执行将跳过审批流程，命令将立即下发到目标设备。此操作会被完整记录。"
   - 出现"紧急原因"必填字段
   - 提交按钮变为红色 "🚨 立即执行"
3. 填写紧急原因（必填，不能为空）
4. 点击"立即执行"后：
   - 系统验证用户有 `command:emergency` 权限
   - 若无权限 → 提示"您没有紧急执行权限，请联系管理员"
   - 若有权限 → 命令立即下发执行
5. **同时**（并行，不阻塞执行）：
   - 向审批策略中配置的审批人发送**通知卡片**（非审批卡片，仅告知）
   - 写入完整审计日志

```
┌─────────────────────────────────────────┐
│  新建命令任务                              │
│                                          │
│  ... (其他字段) ...                        │
│                                          │
│  ☑ 🚨 紧急执行 (跳过审批，直接下发)         │
│     ⚠️ 紧急执行将跳过审批流程，命令将立即     │
│     下发到目标设备。此操作会被完整记录。       │
│                                          │
│     紧急原因: [生产 nginx 502，需立即重启]   │
│     (必填)                                │
│                                          │
│  [取消]              [🚨 立即执行]          │
└─────────────────────────────────────────┘
```

#### 3.4.4 审批人通知

紧急模式下，审批人会收到**两次通知**：

**第一次 — 命令下发时（立即）**：
- 通知类型：🚨 紧急命令通知（红色标题）
- 内容：操作人、命令、目标设备、风险等级、紧急原因
- 说明："此命令已通过紧急通道直接下发执行"
- 无操作按钮（仅通知，无需审批）

**第二次 — 执行结果出来后**：
- 通知类型：✅/❌ 紧急命令执行完成
- 内容：执行结果、输出内容、耗时
- 标注"紧急通道"标签

#### 3.4.5 安全保障

| 保障措施 | 说明 |
|---------|------|
| 权限控制 | 只有拥有 `command:emergency` 权限的用户才能使用 |
| 强制填写原因 | 紧急原因不能为空 |
| 完整审计存档 | 操作人、命令、设备、原因、执行结果全部记录 |
| 实时通知审批人 | 命令下发时立即通知上级，确保知情 |
| 结果二次通知 | 执行结果出来后再次通知上级 |
| 审计报表 | 管理员可在审计事件页面筛选查看所有紧急下发记录 |
| 权限可收回 | 管理员可随时收回 `command:emergency` 权限 |

#### 3.4.6 其他免审批场景

除紧急模式外，还有两种情况命令会跳过审批：

| 场景 | 条件 | 说明 |
|------|------|------|
| 特权用户 | 用户拥有 `command:bypass_approval` 权限 | 超级管理员/SRE 负责人，所有命令自动通过 |
| 策略自动通过 | 审批策略配置了 auto_approve = true | 通常用于 L1 低风险命令 |

### 3.5 IM 集成管理

IM 集成分为**三层配置**，分别由不同角色在不同位置配置。

#### 3.5.1 第①层：IM 渠道配置（管理员）

**入口**：控制台 → 设置 → 集成管理 Tab

**权限**：`integration:manage`（管理员）

**功能**：配置飞书/企业微信/钉钉等 IM 工具的应用凭据，使系统能够向 IM 发送消息。

**交互流程**：

1. 管理员进入"设置 → 集成管理"
2. 页面展示已配置的 IM 渠道列表，每条显示：
   - IM 类型（飞书/企微/钉钉）+ 图标
   - 名称（如"生产环境飞书机器人"）
   - 连接状态（✅ 已连接 / ⚠️ 未配置 / ❌ 连接失败）
   - App ID（脱敏显示）
   - 操作按钮：[测试连接] [编辑] [删除]
3. 点击"+ 添加渠道"，弹出表单：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 渠道类型 | 下拉 | 是 | 飞书 / 企业微信 / 钉钉 / Slack |
| 名称 | 文本 | 是 | 渠道的描述性名称 |
| App ID | 文本 | 是 | IM 开放平台的应用 ID |
| App Secret | 密码 | 是 | IM 开放平台的应用密钥（加密存储） |
| Webhook URL | URL | 否 | 消息推送地址（部分 IM 需要） |

4. 填写完成后，可点击"测试连接"验证凭据是否正确
5. 保存后，该渠道可用于发送审批通知

**安全说明**：
- App Secret 使用 AES-256-GCM 加密存储
- API 响应中不返回 Secret 明文，仅显示脱敏后的 App ID
- 编辑时 Secret 字段显示为 `••••••••`，不修改则保留原值

#### 3.5.2 第②层：用户 IM 绑定（每个用户）

**入口**：控制台 → 设置 → 通知渠道 Tab

**权限**：任何已登录用户（管理自己的绑定）

**功能**：用户将自己的平台账号与 IM 账号关联，用于接收审批通知和命令执行结果。

**交互流程**：

1. 用户进入"设置 → 通知渠道"
2. 页面展示已绑定的 IM 渠道列表：
   - IM 类型 + 图标
   - IM 用户名
   - 验证状态（✅ 已验证 / ⏳ 待验证）
   - 绑定时间
   - 操作按钮：[解绑]
3. 点击"+ 绑定新渠道"，弹出表单：
   - 选择渠道：下拉列表（仅显示管理员已配置的 IM 渠道）
   - 填写 IM 用户 ID
   - 点击"发送验证码" → 系统通过 IM 向该用户发送验证码
   - 输入验证码 → 确认绑定

**绑定方式汇总**：

| 方式 | 操作 | 适用场景 |
|------|------|---------|
| 控制台手动绑定 | 设置 → 通知渠道 → 绑定新渠道 → 验证码 | 标准流程 |
| IM Bot 自助绑定 | 飞书群发 `/bindme email@co.com` → `/verify 123456` | 最简单，推荐 |
| 管理员代绑 | 管理员在用户管理中为用户配置 | 批量初始化 |

**重要说明**：
- 用户 IM 绑定是**接收通知的前提**。未绑定 IM 的审批人只能在控制台"待审批"页面审批，无法收到 IM 通知。
- 一个用户可以绑定多个 IM 渠道（如同时绑定飞书和企微），系统按优先级选择渠道发送通知。

#### 3.5.3 第③层：审批策略（管理员配置审批人）

**入口**：控制台 → 侧边栏 → 审批策略

详见 §3.3 审批策略管理。

### 3.6 IM 审批卡片

#### 3.6.1 审批请求卡片

当命令任务需要审批时，系统向审批人的 IM 发送交互式审批卡片。

**卡片内容**：

```
┌──────────────────────────────────┐
│  🔧 EnvNexus 命令审批请求          │
│  ─────────────────────────────── │
│  申请人: 王五 (运维工程师)          │
│  目标设备: device-001 (共 1 台)    │
│  命令: systemctl restart nginx    │
│  风险等级: L2 (中) 🟡              │
│  备注: 紧急修复，nginx 502 错误     │
│  提交时间: 2026-03-31 14:30       │
│                                   │
│  [✅ 通过]        [❌ 拒绝]         │
└──────────────────────────────────┘
```

**交互逻辑**：
- 审批人点击 [✅ 通过] → 系统回调 Platform API → 任务通过 → 命令下发
- 审批人点击 [❌ 拒绝] → 系统回调 Platform API → 任务拒绝 → 通知运维
- 审批完成后，卡片更新为已处理状态（按钮变灰，显示审批结果）
- 幂等性：同一任务只能审批一次，重复点击无效

#### 3.6.2 执行结果卡片

命令执行完成后，系统向运维人员和审批人发送结果通知。

**成功**：

```
┌──────────────────────────────────┐
│  ✅ 命令执行完成                    │
│  ─────────────────────────────── │
│  任务: 重启生产环境 nginx          │
│  设备: device-001                 │
│  结果: ✅ 成功 (耗时 1.2s)         │
│  输出: nginx.service restarted    │
└──────────────────────────────────┘
```

**失败**：

```
┌──────────────────────────────────┐
│  ❌ 命令执行失败                    │
│  ─────────────────────────────── │
│  任务: 重启生产环境 nginx          │
│  设备: device-002                 │
│  结果: ❌ 失败 (退出码: 1)          │
│  错误: nginx.service not found    │
└──────────────────────────────────┘
```

#### 3.6.3 控制台审批与 IM 审批的关系

审批人可以在**任一渠道**完成审批，两者等价：

| 审批渠道 | 操作方式 | 说明 |
|---------|---------|------|
| 控制台 | 待审批页面 → 点击通过/拒绝 | 适合在电脑前工作时 |
| IM 卡片 | 飞书/企微/钉钉卡片 → 点击通过/拒绝 | 适合移动端、不在电脑前时 |

- 在任一渠道审批后，另一渠道自动同步状态（IM 卡片按钮变灰，控制台列表移除）
- 不会出现重复审批的情况（后端幂等性保证）

## 4. 通知机制

### 4.1 通知事件总览

| 事件 | 通知对象 | 通知渠道 | 说明 |
|------|---------|---------|------|
| 任务创建（需审批） | 审批人 | IM 审批卡片 + 控制台待审批 | 审批人可在任一渠道操作 |
| 任务被通过 | 运维人员 | IM 通知 | 告知命令即将执行 |
| 任务被拒绝 | 运维人员 | IM 通知 | 附带拒绝原因 |
| 任务超时过期 | 运维人员 | IM 通知 | 提示重新提交 |
| 命令执行完成 | 运维人员 + 审批人 | IM 结果卡片 | 附带执行输出 |
| 紧急下发（命令下发时） | 审批人 | IM 紧急通知卡片 | 仅通知，无需操作 |
| 紧急下发（执行结果） | 审批人 | IM 结果卡片 | 标注"紧急通道" |

### 4.2 通知降级策略

| 场景 | 系统行为 |
|------|---------|
| 审批人已绑定 IM | 通过 IM 发送通知 + 控制台待审批可见 |
| 审批人未绑定 IM | 仅控制台待审批可见，创建任务时提示运维人员"审批人未绑定 IM，请通知其登录控制台审批" |
| 未配置任何 IM 渠道 | 所有审批均在控制台进行，无 IM 通知 |
| IM 发送失败 | 记录错误日志，不影响审批流程（审批人仍可在控制台操作） |

## 5. 状态机

### 5.1 命令任务状态流转

```
                    ┌──────────────┐
                    │   创建任务    │
                    └──────┬───────┘
                           │
                    ┌──────▼───────┐
              ┌─────┤ pending_approval │◄─── 需要审批
              │     └──────┬───────┘
              │            │
    ┌─────────▼──┐  ┌──────▼───────┐  ┌──────────┐
    │  cancelled  │  │   approved   │  │  denied   │
    │  (运维取消) │  │  (审批通过)   │  │ (审批拒绝) │
    └────────────┘  └──────┬───────┘  └──────────┘
                           │
                    ┌──────▼───────┐
                    │  executing   │◄─── 命令下发中
                    └──────┬───────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
       ┌──────▼──┐  ┌──────▼──┐  ┌──────▼──────┐
       │completed │  │ failed  │  │partial_done │
       │ (全部成功)│  │(全部失败)│  │ (部分成功)   │
       └─────────┘  └─────────┘  └─────────────┘

    特殊路径:
    ─ 自动通过 (auto_approve): 创建 → approved → executing → ...
    ─ 紧急模式 (emergency):    创建 → approved → executing → ...
    ─ 审批超时:                pending_approval → expired
```

### 5.2 状态流转规则

| 当前状态 | 可转移到 | 触发条件 |
|---------|---------|---------|
| pending_approval | approved | 审批人通过 |
| pending_approval | denied | 审批人拒绝 |
| pending_approval | cancelled | 运维人员取消 |
| pending_approval | expired | 超时自动过期 |
| approved | executing | 命令开始下发 |
| executing | completed | 所有设备执行成功 |
| executing | failed | 所有设备执行失败 |
| executing | partial_done | 部分设备成功，部分失败/跳过 |

**约束**：状态流转是单向的，不可逆。

## 7. 异常处理与边界条件

### 7.1 设备相关

| 场景 | 系统行为 |
|------|---------|
| 目标设备离线 | 该设备的执行记录标记为 `skipped`，其他在线设备正常执行 |
| 所有目标设备离线 | 任务状态变为 `failed`，通知运维人员 |
| 命令执行超时 | 该设备执行记录标记为 `timeout`（默认超时 60 秒，可在命令中配置） |
| 设备在执行过程中断开 | 执行记录标记为 `failed`，错误信息为"设备连接中断" |

### 7.2 审批相关

| 场景 | 系统行为 |
|------|---------|
| 审批人账号被禁用 | 任务无法审批，运维人员需联系管理员更换审批人或取消任务 |
| 审批策略被删除（任务已创建） | 已创建的任务不受影响，仍按原审批人处理 |
| 同一任务被多人审批（角色审批） | 第一个审批的人生效，后续审批操作返回"已处理" |
| IM 回调签名验证失败 | 拒绝处理，记录安全日志 |

### 7.3 并发与幂等

| 场景 | 系统行为 |
|------|---------|
| 审批人在控制台和 IM 同时点击通过 | 后端幂等处理，仅第一次生效 |
| 运维人员重复提交相同命令 | 允许创建多个任务（每次是独立任务） |
| 审批超时与审批操作同时发生 | 以先到达的为准（乐观锁） |

## 9. 审计与合规

### 9.1 审计事件类型

| 事件类型 | 说明 |
|---------|------|
| `command.task_created` | 命令任务创建 |
| `command.task_approved` | 命令任务被通过 |
| `command.task_denied` | 命令任务被拒绝 |
| `command.task_cancelled` | 命令任务被取消 |
| `command.task_expired` | 命令任务审批超时 |
| `command.execution_completed` | 命令执行完成（含结果） |
| `command.emergency_bypass` | 紧急模式跳过审批（额外记录原因） |
| `approval_policy.created` | 审批策略创建 |
| `approval_policy.updated` | 审批策略修改 |
| `approval_policy.deleted` | 审批策略删除 |
| `im_provider.created` | IM 渠道配置创建 |
| `im_provider.updated` | IM 渠道配置修改 |

### 9.2 紧急模式审计

紧急模式的审计记录包含额外字段：

| 字段 | 说明 |
|------|------|
| bypass_reason | 紧急原因（运维人员填写） |
| bypass_type | 跳过类型：emergency / privileged_user / auto_policy |
| approver_notified | 被通知的审批人 |
| notification_channels | 通知渠道（feishu / wechat_work / dingtalk） |

管理员可在控制台"审计事件"页面通过筛选 `event_type = command.emergency_bypass` 查看所有紧急下发记录，监控是否被滥用。

## 10. 完整用户旅程

### 10.1 旅程一：标准审批流程

```
