# EnvNexus 商业产品白皮书

> **文档版本**：v1.0
> **产品名称**：环枢 (EnvNexus)
> **产品定位**：AI 原生的企业级环境治理、安全诊断与引导式修复平台

---

## 1. 产品愿景与定位

在现代企业 IT 环境中，终端用户的运行环境（操作系统、网络、依赖库、配置）日益复杂。传统的远程支持工具（如 TeamViewer、AnyDesk）虽然能解决问题，但赋予了支持人员**无限制的系统控制权**，带来了极大的安全合规风险。

**EnvNexus（环枢）** 致力于重塑企业 IT 支持与环境治理的边界。我们提供一个**“默认只读、审批执行、全量审计”**的 AI 驱动平台。它既能利用大模型的强大推理能力快速定位环境故障，又能通过严格的本地策略和用户审批机制，确保每一次系统变更都处于绝对的安全管控之下。

**一句话定义**：
`EnvNexus 是一套在零信任架构下，通过 AI 辅助实现终端环境安全诊断与审批式修复的治理中枢。`

---

## 2. 核心痛点与解决方案

### 2.1 行业痛点
1. **信任危机**：传统远控软件权限过大，IT 人员可以随意浏览用户隐私文件或植入后门。
2. **排障成本高**：复杂的环境问题（如环境变量冲突、端口占用、依赖缺失）往往需要高级工程师耗费大量时间排查。
3. **合规审计缺失**：终端上的修复操作（如修改了哪个注册表、删除了哪个文件）通常没有结构化的操作日志，事后无法追责和回滚。
4. **SaaS 交付难**：不同企业客户对数据隐私要求不同，难以用单一的公有云产品满足所有企业的合规要求。

### 2.2 EnvNexus 解决方案
1. **诊断与修复分离**：系统默认只拥有“只读”权限进行环境扫描。任何实质性的修改动作（如重启服务、修改配置）必须转化为一个“审批请求”（Approval Request）。
2. **AI 智能赋能**：内置 LLM 路由，自动分析系统日志、进程状态和网络配置，给出人类可读的故障原因和修复建议。
3. **终端用户赋权**：修复动作下发到终端后，必须由终端用户在图形界面上点击“同意”，底层内核才会真正执行。
4. **全链路结构化审计**：从诊断发起、AI 建议、用户审批到最终执行结果，所有环节均生成不可篡改的结构化审计日志。

## 3. 产品最终形态

EnvNexus 采用 **“云端大脑 + 瘦客户端 + 强执行内核”** 的三位一体产品形态：

### 3.1 EnvNexus Platform (云端治理中枢)
*   **形态**：Web 控制台 (SaaS / 私有化部署)。
*   **功能**：
    *   **多租户管理**：支持多企业组织隔离。
    *   **策略配置中心 (Profiles)**：管理员在此配置 AI 模型接入（Model Profile）、安全拦截策略（Policy Profile）和客户端行为（Agent Profile）。
    *   **分发管理**：一键生成租户专属的客户端下载链接（支持免填参数的静默激活）。
    *   **全局审计台**：查看所有终端的诊断记录、审批流水和变更历史。

### 3.2 EnvNexus Agent Desktop (终端交互界面)
*   **形态**：跨平台桌面应用程序（Windows / macOS / Linux），常驻系统托盘。
*   **功能**：
    *   **用户交互窗口**：提供类似对话框的 UI，用户可在此用自然语言描述问题（如“我的 Docker 启动不了了”）。
    *   **审批工作台**：弹出清晰的变更确认框，展示即将执行的底层命令及其风险等级。
    *   **状态看板**：展示当前设备的安全基线状态和平台连接状态。
    *   *注：Desktop 仅为展示层，不包含任何提权或系统修改能力。*

### 3.3 EnvNexus Agent Core (本地执行内核)
*   **形态**：无 UI 的底层守护进程（Go 编写的独立二进制文件）。
*   **功能**：
    *   **沙箱执行器**：真正执行网络探测、进程读取、文件修改的引擎。
    *   **本地策略拦截网关**：即使云端 AI 建议删除某文件，如果本地策略（Policy）禁止，Core 会直接在底层拦截该操作，不予展示审批。
    *   **离线降级模式**：在断网情况下，仍能提供基础的只读诊断和日志打包导出功能。

## 4. 核心业务场景 (Use Cases)

### 场景一：员工自助 IT 支持 (Self-Service)
1. 员工遇到开发环境报错，打开桌面端输入错误信息。
2. Agent Core 自动采集报错相关的系统上下文。
3. 云端 AI 分析后，下发一条“清理 DNS 缓存并重启网卡”的修复建议。
4. 员工点击“同意”，问题解决，全过程无需 IT 人员介入。

### 场景二：远程专家协助 (Remote Guided Repair)
1. 专家在云端控制台选中某台在线设备，发起远程诊断会话。
2. 专家通过云端下发诊断脚本，获取设备状态。
3. 专家编写一段修复脚本（如修改 Nginx 配置）并下发。
4. 终端桌面弹出提示：“IT 专家申请修改 Nginx 配置，是否允许？”。
5. 员工确认后，脚本执行，结果回传给专家。

### 场景三：企业级批量合规治理 (Enterprise Governance)
1. 管理员在云端配置“禁止开启 SSH 密码登录”的基线策略。
2. 所有终端 Agent 定期进行本地扫描（Governance Scan）。
3. 发现违规设备后，自动生成修复工单并推送到终端桌面等待用户确认。

## 5. 商业化与交付模式

EnvNexus 提供灵活的商业交付模式，以适应不同规模企业的合规要求：

1.  **SaaS 托管版 (Hosted)**：
    *   开箱即用，按设备并发连接数或活跃设备数（MAU）按月订阅计费。
    *   适合中小企业和初创团队。
2.  **混合云版 (Hybrid)**：
    *   控制台和信令网关由 EnvNexus 托管。
    *   **模型与数据驻留本地**：客户可配置使用企业内部的私有大模型，且敏感审计日志直接写入客户的私有 S3 存储，不经过公有云。
3.  **完全私有化部署 (Private)**：
    *   提供完整的 Kubernetes Helm Chart，客户在内网完全独立部署。
    *   通过 License Key 进行商业授权激活。
    *   适合金融、军工、政府等强监管行业。

## 6. 用户旅程 (User Journey)

1.  **入驻**：企业 IT 管理员注册 EnvNexus 云平台，创建租户。
2.  **配置**：配置企业专属的 OpenAI 密钥和安全策略。
3.  **分发**：管理员在后台点击“生成下载链接”，将链接发送给全员。
4.  **安装**：员工下载客户端（安装包已在云端动态注入了该企业的身份信息），双击安装后**自动完成绑定**，无需输入任何激活码。
5.  **日常使用**：客户端静默运行。遇到问题时唤起界面进行诊断。
6.  **审计与复盘**：IT 部门每月通过云端控制台导出脱敏后的审计报表，用于合规检查。
