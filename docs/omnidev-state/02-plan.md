# EnvNexus 客户端在线状态优化计划

## 任务目标
优化客户端已退出但显示在线的问题，确保设备状态能够及时准确地反映客户端的实际状态。

## 任务分组

### 1. 分析与设计
- [x] 分析当前心跳机制和在线状态判断逻辑
- [x] 设计优化方案

### 2. 后端优化
- [ ] 修改 Session Gateway，在 WebSocket 连接断开时更新设备状态
- [ ] 优化设备状态管理逻辑，确保连接断开时及时更新 last_seen_at

### 3. 前端优化
- [ ] 调整前端在线状态判断逻辑，结合连接状态和时间戳
- [ ] 优化轮询机制，提高状态更新的实时性

### 4. 客户端优化
- [ ] 缩短客户端心跳间隔，从 1 小时改为 5 分钟
- [ ] 实现优雅退出机制，确保退出时发送离线通知

## 执行顺序
1. 后端优化（Session Gateway 修改）
2. 前端优化（状态判断逻辑）
3. 客户端优化（心跳间隔和退出机制）

## 预期结果
- 客户端退出后，设备状态能够在 90 秒内更新为离线
- 在线状态判断更加准确，避免误显示
- 设备状态更新的实时性得到显著提升
